# Plano de Testes – Login e Cadastro

## 1. Introdução
Este documento descreve o plano de testes para validação das funcionalidades
de Login e Cadastro de Usuário em uma aplicação web, com foco na garantia
da qualidade, validação funcional e regras de negócio.

## 2. Objetivo
Garantir que as funcionalidades de Login e Cadastro funcionem conforme os
requisitos definidos, prevenindo falhas em produção e assegurando uma boa
experiência ao usuário final.

## 3. Escopo dos Testes
### 3.1 Funcionalidades testadas
- Cadastro de usuário
- Login
- Validação de campos obrigatórios
- Mensagens de erro
- Autenticação via API

### 3.2 Fora do escopo
- Testes de performance
- Testes de segurança
- Testes de carga

## 4. Tipos de Teste
- Testes funcionais
- Testes manuais
- Testes exploratórios
- Testes de API
- Testes de regressão

## 5. Ambiente de Testes
- Aplicação web
- Navegador: Google Chrome
- Sistema operacional: Windows / Linux / macOS

## 6. Critérios de Entrada
- Requisitos definidos
- Ambiente disponível
- Casos de teste escritos

## 7. Critérios de Saída
- Casos de teste executados
- Bugs críticos identificados e reportados
- Evidências coletadas

## 8. Riscos
- Requisitos incompletos
- Ambiente instável
- Falta de dados de teste
